  <action>
    1. Create `Sources/KeychainService.swift` as an `enum KeychainService` (no instances needed) with:
       - `private static let service = "com.claumagotchi.apikeys"`
       - `static func save(_ value: String, account: String)` — if value is empty, call delete() instead. Otherwise try SecItemAdd; on errSecDuplicateItem, SecItemUpdate. Always set `kSecAttrSynchronizable: false` to prevent iCloud sync.
       - `static func load(account: String) -> String?` — SecItemCopyMatching with kSecReturnData. Return nil if not found or empty.
       - `static func delete(account: String)` — SecItemDelete.
       - Account constants: `static let groqAccount = "groq"` and `static let openAIAccount = "openai"`.

    2. Create `Sources/GroqTranscriptionService.swift` as an `enum GroqTranscriptionService` with:
       - `static func transcribe(wavURL: URL, apiKey: String) async throws -> String`
       - POST to `https://api.groq.com/openai/v1/audio/transcriptions`
       - Authorization header: `"bearer \(apiKey)"` (lowercase bearer — Groq requirement, NOT "Bearer")
       - Multipart form-data body with boundary UUID:
         - Field "model" = "whisper-large-v3-turbo"
         - Field "file" = WAV data from wavURL, filename "voice.wav", content-type "audio/wav"
       - Parse response JSON: `{ "text": "..." }` using Decodable struct
       - Delete temp WAV file after successful upload: `try? FileManager.default.removeItem(at: wavURL)`
       - On HTTP error, throw a descriptive error (include status code)

    3. Create `Tests/ClaumagotchiTests/KeychainServiceTests.swift`:
       - `testKeychainSaveAndLoad()` — save "test-key-123" to account "test-groq", load returns "test-key-123", then delete
       - `testKeychainEmptyDeletesBehavior()` — save "abc" then save "" → load returns nil
       - `testKeychainUpsert()` — save "v1" then save "v2" → load returns "v2"
       - `testKeychainDeleteNonexistent()` — delete non-existent account does not crash
       - Use a test-specific account name like "test-groq" to avoid polluting real keys
       - Clean up in tearDown: delete test accounts

    Note: `@testable import` is NOT available for .executableTarget in this project (Phase 13 decision). The KeychainService enum must be accessible without @testable — since it's in the same module/target, XCTest can still access internal types. However, if the test target has no dependency on the executable target, you may need to duplicate the KeychainService logic in tests or test via integration. Check Package.swift test target configuration.
