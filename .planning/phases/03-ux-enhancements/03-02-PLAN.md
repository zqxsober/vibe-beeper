4. In `deinit`, add cleanup for both monitors (before or after the existing cleanup):
   ```swift
   if let m = globalKeyMonitor { NSEvent.removeMonitor(m) }
   if let m = localKeyMonitor { NSEvent.removeMonitor(m) }
   ```

5. Add `setupGlobalHotkeys()` method after the existing `startIdleTimer` method:
   ```swift
   private func setupGlobalHotkeys() {
       guard globalKeyMonitor == nil, AXIsProcessTrusted() else { return }
       globalKeyMonitor = NSEvent.addGlobalMonitorForEvents(matching: .keyDown) { [weak self] event in
           self?.handleHotKey(event)
       }
       localKeyMonitor = NSEvent.addLocalMonitorForEvents(matching: .keyDown) { [weak self] event in
           self?.handleHotKey(event)
           return event
       }
   }
   ```

   Key design decisions:
   - `globalKeyMonitor == nil` guard prevents double-installation
   - `AXIsProcessTrusted()` returns false until user grants Accessibility — monitor is silently not installed
   - Global monitor fires for OTHER apps, local monitor fires when companion window has focus
   - Local monitor MUST return the event (non-nil) to allow normal event processing to continue

6. Add `handleHotKey(_:)` method immediately after `setupGlobalHotkeys()`:
   ```swift
   private func handleHotKey(_ event: NSEvent) {
       guard pendingPermission != nil else { return }
       let flags = event.modifierFlags.intersection(.deviceIndependentFlagsMask)
       guard flags == .option else { return }
       switch event.keyCode {
       case 0:  // kVK_ANSI_A
           DispatchQueue.main.async { self.respondToPermission(allow: true) }
       case 2:  // kVK_ANSI_D
           DispatchQueue.main.async { self.respondToPermission(allow: false) }
       default:
           break
       }
   }
   ```

   Key design decisions:
   - `pendingPermission != nil` gate ensures hotkeys are no-ops when no permission is pending (avoids terminal Meta key conflicts)
   - `flags == .option` ensures ONLY Option is held — not Cmd+Option, not Ctrl+Option, not Shift+Option
   - `DispatchQueue.main.async` ensures respondToPermission runs on main thread (global monitor callback may not be on main)
   - keyCode 0 = A, keyCode 2 = D (hardware keycodes, layout-independent)

7.
